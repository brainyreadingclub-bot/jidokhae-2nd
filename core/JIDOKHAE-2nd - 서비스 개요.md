# JIDOKHAE 서비스 개요

> 이 문서는 AI 코딩 에이전트가 개발 시 참고하는 최상위 맥락 문서입니다.
> "지금 뭘 만들어야 하는가(MVP)"와 "왜 만드는가, 어디로 가는가(비전)"를 모두 담고 있습니다.
> 애매한 판단이 필요할 때 이 문서의 본질과 사용원칙으로 돌아와 검토하세요.

## 1. 서비스 정의

**프로젝트명:** jidokhae-2nd  
**서비스명:** JIDOKHAE  
**한 문장 정의:** 지독해 회원이 모임 일정을 한눈에 보고, 바로 신청하고, 간편하게 결제하는 곳

---

## 2. 왜 만드는가

### 2-1. 현재 상황

지독해는 경주/포항에서 운영되는 독서모임이다. 250명의 회원이 카카오톡 오픈채팅방 안에서 일정 확인, 참가 신청, 결제를 처리하고 있다.

현재 운영 방식:
- 모임 일정 공지 → 오픈카톡방
- 참가 신청 → 댓글로 "신청 완료"
- 결제 → 입금자명 변경 후 계좌이체
- 입금 확인 → 운영자가 수동으로 확인
- 취소/환불 → 개인 연락 → 운영자가 입금 내역 확인 후 처리

### 2-2. 해결하려는 문제

1. **회원이 모임 일정을 쉽게 확인할 수 없다** — 일정이 대화에 묻히고, 한눈에 들어오지 않는다
2. **신청 과정이 번거롭다** — 입금자명 변경 → 계좌이체 → 댓글 신청의 3단계를 거쳐야 한다
3. **결제 과정이 불편하다** — 계좌이체 → 인증 → 운영자 수동 확인까지 시간이 걸린다
4. **정원이 찼을 때 대기할 방법이 없다** — 마감되면 기다릴 수 없고, 자리가 나도 알 수 없다

### 2-3. 동기

일정이 참여하고 싶게 생겨야 한다. 지금은 일정 자체가 눈에 안 들어오고, 봐도 신청이 귀찮아서 미루다가 놓친다. 리마인드를 보낼 때마다 신청이 몰리는 걸 보면, 평소에는 정보에 접근하기 어렵다는 뜻이다. 이 불편함이 참여의 문턱을 높이고, 참여의 문턱을 낮춰야 쉽게 참여한다.

## 3. 본질

### 3-1. 비전 (이 서비스가 궁극적으로 추구하는 것)

> **표면:** 복잡한 프로세스를 간소화하고 싶다
>
> **본질:** 회원들의 부담을 줄여주고, 지독해에 소속감과 자부심을 느끼게 해주고 싶다
>
> "바쁜 현생을 사는 회원들이 부담 없이 자연스럽게 참여하고, 자기가 얼마나 꾸준히 함께했는지 느낄 수 있게"

### 3-2. MVP가 해결하는 범위

비전 중 **"부담 없이 자연스럽게 참여하고"** 부분을 먼저 해결한다. "꾸준히 함께했는지 느낄 수 있게"(소속감, 활동 기록, 배지 등)는 추후 개발한다.

---

## 4. 사용원칙

모든 기능과 화면은 아래 세 가지 원칙을 기준으로 판단한다.

1. **쉬워야 한다** — 3초 안에 일정 확인, 3번 클릭 안에 신청 완료
2. **모여야 한다** — 흩어진 일정과 신청이 한 곳에
3. **보여야 한다** — 운영자는 신청/결제 현황을, 회원은 일정과 내 신청 내역을

## 5. 핵심 가치

### 5-1. 비전 전체의 핵심 가치

| 가치 | 설명 | MVP 해당 |
|------|------|----------|
| **쉬운 연결** | 일정 확인부터 신청/결제까지 최소한의 동작으로 | ✅ |
| **신뢰** | 결제가 투명하고, 내 신청 상태가 명확하게 보임 | ✅ |
| **잊지 않게** | 자동 리마인드로 모임과의 연결 유지 | ❌ 추후 |
| **소속감** | 활동 기록, 배지, 칭찬을 통해 "나는 지독해 회원"이라는 느낌 | ❌ 추후 |

### 5-2. MVP가 집중하는 가치

**쉬운 연결**과 **신뢰**. 회원이 일정을 보고 참여하고 싶어서 바로 누르고, 결제까지 매끄럽게 끝나는 경험.

## 6. 타겟 사용자

### 6-1. 비전 전체의 타겟

| 구분 | 대상 | 설명 |
|------|------|------|
| 주 사용자 | 기존 회원 | 정기모임에 참여하거나 참여하려는 회원. 바쁜 일상 속에서 독서모임을 유지하고 싶은 사람 |
| 부 사용자 | 신규 회원 | 지독해를 처음 알게 된 사람. 후킹 랜딩페이지를 통해 유입 |
| 운영자 | 영탁(단무지) | 모임 운영, 정산, 회원 관리. 다른 운영진에게 일부 권한 위임 가능 |

### 6-2. MVP의 타겟

| 구분 | 대상 | 설명 |
|------|------|------|
| 주 사용자 | 기존 회원 | 카카오 로그인으로 접속하여 일정 확인, 신청, 결제하는 사람 |
| 운영자 | 영탁(단무지) | 모임 생성/수정/삭제, 신청자 목록 확인, 결제 상태 확인 |

> MVP에서 신규 회원 후킹 랜딩페이지와 운영자 권한 위임은 제외한다.

## 7. 서비스 규칙

서비스에 반영되어야 하는 비즈니스 규칙이다.

### 7-1. 모임 규칙
- 정기모임 장소: 경주(수요일), 포항(일요일)
- 정기모임 정원: 14명
- 정기모임 참가비: 10,000원 (모임별 가변 금액 — 추후 토론모임 25,000원 등 다른 금액이 적용될 수 있으므로 고정값이 아닌 모임별 설정 방식으로 구현)
- 정기모임 시간: 약 2시간

### 7-2. 참여 자격 규칙 (추후 적용)
- 모든 모임 참여의 전제조건 = 정기모임 참여 이력
- 6개월 이내 정기모임 참여 기록 없으면 다른 모임(토론모임, 번개모임 등) 참여 불가

> MVP는 정기모임만 다루므로 자격 규칙은 적용하지 않는다. 단, 추후 다른 모임 유형이 추가될 때 이 규칙이 적용되어야 하므로 참여 이력 데이터는 쌓아둔다.

### 7-3. 취소/환불 규칙
- MVP: 회원이 직접 취소 → 환불 규정에 따라 PG사 환불 API로 자동 환불
- 환불 기준 (정기모임): 3일 전 100% / 2일 전 50% / 이후 환불 불가 (취소 자체는 가능, 환불 금액 0원)
- 운영자가 모임 삭제 시: 모든 신청자에게 100% 자동 환불

### 7-4. 대기 규칙 (추후 적용)
- MVP에서 대기 신청은 제외
- 추후: 대기 신청 → 자동 승격 → 결제 유도 구조로 구현 예정

## 8. MVP 핵심 흐름

**핵심 흐름: 일정 확인 → 신청/결제 → 취소/환불**

```
회원이 웹에 접속한다 (카카오 로그인)
    ↓
모임 일정 목록이 한눈에 보인다 (참여하고 싶은 디자인, 마감 모임은 "마감" 표시, 내가 신청한 모임은 "신청완료" 표시)
    ↓
원하는 모임을 누른다
    ↓
모임 상세 정보를 확인한다 (날짜, 시간, 장소, 남은 자리)
    ↓
"신청하기" 버튼을 누른다
    ↓
간편결제 화면으로 이동한다 → 결제 완료 = 신청 확정 (결제 실패 시 모임 상세로 복귀, 안내 메시지 표시)
    ↓
신청 확정 화면 ("신청이 완료되었습니다" + 모임 정보 요약)
    ↓
모임 일정 목록으로 돌아온다 (해당 모임에 "신청완료" 표시)
    ↓
내 신청 내역에서 확인할 수 있다
    ↓
(취소 시) 내 신청 내역 → 해당 모임 상세 → 취소 버튼 → 환불 규정 확인 → 취소 확정 → 자동 환불
```

---

## 9. MVP에 포함되는 것

**회원 기능:**
- 카카오 로그인
- 모임 일정 목록 (한눈에 보이는 디자인, 마감 표시, 내가 신청한 모임 "신청완료" 표시)
- 모임 상세 페이지 (confirmed 신청 유무와 모임 시점에 따라 버튼 분기 — 상세는 PRD 참조)
- 모임 신청 + 간편결제 (결제 완료 = 신청 확정, 취소 후 재신청 가능)
- 신청 확정 화면
- 내 신청 내역 확인 (과거 모임 이력 포함)
- 셀프 취소 + 자동 환불 (환불 규정에 따라 자동 계산, 모임 날짜 다음 날부터 취소 불가)

**운영자 기능:**
- 모임 생성 / 수정 / 삭제
- 신청자 목록 확인
- 결제 상태 확인

## 10. MVP에 포함되지 않는 것 (추후 개발)

아래 항목은 비전에는 포함되지만, 이번 MVP에서는 만들지 않는다. AI 코딩 에이전트는 이 기능들을 MVP에 넣지 않되, 추후 확장 가능하도록 데이터 구조는 고려한다.

- 콩 시스템 (포인트 화폐)
- 대기 신청 / 자동 승격
- 알림톡 / 리마인드
- 배지 / 칭찬 / 활동 기록
- 글쓰기 챌린지 / 필사 / 독서 인증
- 소개 랜딩페이지 (SEO / 후킹)
- 관리자 대시보드 (인사이트 / 통계)
- 운영진 협업 도구 / 권한 위임
- 지똑똑 AI 챗봇 연동
- 내 책장 (읽은 책 기록)

## 11. MVP 성공의 모습

이 서비스가 잘 돌아가면 이런 상태가 된다.

**회원 입장:**
- 웹에 들어오면 다가오는 모임 일정이 한눈에 보인다. 일정이 참여하고 싶게 생겼다.
- 마음에 드는 모임을 누르고, 바로 결제까지 끝난다. 카톡에서 입금자명 바꾸고 이체하던 시절이 끝났다.
- 내가 뭘 신청했는지 언제든 확인할 수 있다.
- 취소하고 싶으면 직접 취소하고, 환불도 자동으로 된다. 운영자에게 연락하고 기다릴 필요가 없다.

**운영자 입장:**
- 모임을 등록하면 회원들이 알아서 신청하고 결제한다.
- 신청자 목록과 결제 상태를 한 화면에서 바로 확인할 수 있다. 구글시트와 은행앱을 왔다갔다 할 필요가 없다.
- 입금 확인에 쓰던 시간이 사라진다.
- 환불 요청에 대응하던 시간도 사라진다.

---

## 12. 이전 프로젝트(1st)와의 관계

서비스 철학, 사용원칙 3가지, 해결하려는 문제의 본질은 1st와 동일하다. 1st에서는 소속감(배지, 칭찬, 책장), 자동 환불, 리마인드 알림톡, 운영자 대시보드 등 광범위한 기능을 한 번에 담으려 했으나, 이번 2nd에서는 **핵심 흐름(일정 → 신청/결제 → 취소/환불)**으로 범위를 줄여 실제 출시를 최우선으로 한다.

비전은 1st가 그리는 전체 그림이고, MVP는 그 그림의 첫 번째 조각이다.
