섹션 노출 규칙 없으면 안 그린다

  • 인사 (always) — 닉네임 + 지역(있으면) + 오늘 날짜
  • Hero 한 줄 — 데이터 상태에 따라 카피 변경 (D-Day 있음/모임 있음/비수기)
  • 오늘/내일 모임 (조건부) — 현재 user의 confirmed 중 date IN (오늘, 내일) 있을 때만 표시. 없으면 섹션 자체 미렌더
  • 다가오는 모임 (조건부) — active 모임 중 date >= 오늘 가까운 순 최대 3개. 0건이면 빈 상태 카드 1개 (마음에 드는 한 마디)
  • CTA (always) — "모임 일정 보러가기" — 회원이 막다른 길 안 느끼게

M8 / M9 도착 시 자동 추가: 배너 (banners.is_active=true 데이터 있으면 최상단), 이번 주 한 줄 (book_quotes.status=approved 있으면), 합류 티커 (M9 소셜프루프). 데이터 fetch 코드만 추가하고 빈 처리는 동일 패턴.

구현 메모 files / data / behavior

  • BottomNav — 기존 BottomNav.tsx에 홈 탭 추가 (HomeIcon inline SVG). 3탭 grid 자동 분배
  • 홈 페이지 — src/app/(main)/home/page.tsx Server Component. HomeContent Suspense fallback
  • 데이터 fetch — Promise.all 병렬: ① 오늘/내일 confirmed registrations (현재 user) ② active 모임 가까운 3개
  • fallback 모임 카드 — 기존 MeetingCard 재사용 (간소화 판일 수도 — Mini용 별도 컴포넌트는 만들지 않고 일단 재사용)
  • D-Day 입장권 — M9 정식 카드 component DDayTicket는 M9에서. M7는 인라인 jsx로 충분 (재사용 적음)
  • 기본 랜딩 — 로그인 후 / (모임 일정) 유지. /home은 능동적 진입만
  • 토큰 — 기존 globals.css 토큰만
