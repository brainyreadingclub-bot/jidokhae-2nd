# JIDOKHAE 기술 학습서

> 프로젝트 총괄 파트너를 위한 코드베이스 이해 가이드.
> 읽고 나면 `/full-scan` 보고서를 해독하고, 설계 의사결정에 의견을 낼 수 있다.

---

## 이 책이 필요한 이유

프로젝트가 커지면서 에이전트 리포트와 코드 변경 내역이 점점 난해해졌다.
"RLS", "idempotency", "cron", "RPC" 같은 단어가 쏟아지는데 하나하나 맥락을 물어보는 건 비효율적이다.

이 책은 **한 번 읽고 나면 다음 대화가 편해지도록** 용어·구조·설계 의도를 한 곳에 모은 사전이자 지도다.

**이 책이 다루지 않는 것**

- 코딩 튜토리얼 (내가 직접 코드를 짜기 위한 것이 아님)
- 프레임워크 공식 문서 대체 (Next.js/Supabase 공식 문서와 경쟁하지 않음)
- 최신 트렌드 (2026-04 시점의 이 프로젝트에만 집중)

## 막힐 때

- 용어가 낯설면 우선 [01. 용어사전](part1-기초/01-용어사전.md)에서 `Ctrl+F` 검색.
- 여전히 이해 안 되면 Claude에게 **"용어사전 XX 섹션 비유로 다시 설명해줘"** 라고 요청.
- 챕터가 어렵다면 Claude에게 **"이 챕터를 초등학생도 알아듣게 예시 3개 추가해서 설명해줘"** 라고 요청.

이 책은 살아있는 문서다. 읽다가 이해 안 가는 부분을 표시해두고 세션 말미에 말해주면, Claude가 해당 섹션을 보강한다.
