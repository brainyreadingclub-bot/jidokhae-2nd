## 라운드 1: 독립 의견

### 🎨 김도윤
**핵심 주장:** 페어링과 팔레트는 시대를 잡았으나, 여백/사이즈/라운드 디시플린이 무너져 "잘 만든 노션 페이지" 수준에 머문다.

**근거:** `--bg:#F7F4EC` + 22px dot grid + Instrument Serif italic으로 숫자/라벨 잡고 Noto Serif KR로 본문 한글 헤드 누르는 페어링은 정석. 그러나 타입 사이즈가 `10.5/11/11.5/12/12.5/13/13.5/14.5px`로 0.5px 단위 임의 조정 30곳, border-radius도 `5/6/8/10/14/18/42px` 7개로 산만 — `--r-sm/md/lg` 정의해놓고 본문에서 거의 안 씀. `.mcard::before width:3px`와 `.qitem border-left:2px`처럼 같은 의미의 좌측 색상 바 두께가 다름. **시스템을 만들어놓고 안 따른다.**

**리스크:** 이 mockup이 jidokhae-web으로 토큰 옮길 때 거짓 레퍼런스가 됨 — M11 "토큰 100% 적용"이 오히려 후퇴.

**추천:**
1. 타입 스케일 8단계로 강제 통일 (`--fs-xs:11 / sm:12 / base:13 / md:14 / lg:17 / xl:24 / 2xl:30 / display:88`). 0.5px 전부 스냅
2. 라운드 토큰 3단계로 압축 (`--r-sm:6 / md:10 / lg:14`)
3. 세로 리듬 8px 그리드 강제 (`8/16/24/32/48/64/96`만 사용)
4. DIFF 셀당 strong 1개 룰 — 현재 persimmon 단어 도배로 핵심이 묻힘
5. 한글 본문 폰트 무게 1단계 다운 — Noto Serif KR 700+ 도배 → 카드 헤드 500~600, hero/sec-head만 800~900

### 🧭 박서연
**핵심 주장:** 비교 문서 시각 완성도는 높지만 "30초 진단"이 어렵다 — 가장 큰 결정 메시지(M9·M10 병렬)가 마지막에 작은 글씨로 묻혀 있다.

**근거:** Hero는 감성 카피만 전달하고 정량 정보(WP 19, Scenario 138, M7~M12)는 우측 aside 12~13px 회색으로 종속. Section A의 3-col compare는 "Before 1 vs After 2"라는 비대칭에 시각적 단서 없어 한 박자 늦게 인지됨. Insight strip Now/Next는 동일 사이즈여서 "해법이 무엇을 더 주는가"가 한눈에 안 들어옴. **가장 중요한 인사이트인 "M10은 M7만 의존 → M9와 병렬 가능"이 footer 직전 14px italic 한 줄에 묻힘** — 이걸 보고 "Phase 3 기간 단축"이라는 의사결정을 끌어내기 어려움.

**리스크:** "오너 1인 평가용"이라는 목적 대비 "보기 좋은 잡지"에 가깝게 만들어짐 — 결정 포인트가 평문으로 흩어짐.

**추천:**
1. Hero를 결정형으로 — "19 WP · 6 Milestones · M9·M10 병렬 시 N주 단축 가능" 같은 의사결정 숫자 3개를 48~64px로
2. Section A 비대칭 명시화 — After 2장 위에 "After — 홈 + 모임" 공통 헤더 띠
3. Insight strip 비대칭 비율 (Now 1fr : Next 1.4fr) + Next에 변화량 배지 (`+4 섹션`, `+1 탭`)
4. Timeline 병렬 정보 시각화 승격 — M9~M10 사이 점선 화살표 + "PARALLEL" 배지 + 가능하면 mini Gantt 1줄
5. DIFF 매트릭스에 마일스톤 태그 (`M9` `M10` 등) — 표를 읽으며 "언제 도착하나" 즉시 매핑

### 🎯 이진우
**핵심 주장:** "매거진B 표지처럼 보이려다 만든 인디 굿즈 카탈로그." 톤은 30+ 큐레이터에게 자연스럽고, 정작 25세 회원과 어긋남.

**근거:** 폰트 스택(Instrument Serif italic + Noto Serif KR 900 + JetBrains Mono)과 카피("Editorial Brief No. 03", "Now showing", "indie editorial", "指讀解")는 매거진B/롱블랙 톤이지 29CM/무신사매거진 톤이 아님. 29CM 카피는 훨씬 짧고 구어체("이 책, 지금 너 책상에 있어야 해"), 무신사매거진은 사진+짧은 캡션 위주로 세리프를 거의 안 씀. H1 88px의 `<em>머무는 곳</em>` italic + 어른스러운 lede("머물 이유를, 호흡할 공간을")는 25세보다 35세가 자기 자신을 투영하기 쉬운 톤 — **직전 피드백이 지적한 그 지점에서 변하지 않음.** 감빛은 좋지만 모스 그린 + 도트 그리드 + 1.5px 잉크 보더가 합쳐지며 '레트로 인쇄소' 무드로 다시 끌어내림.

**리스크:** "독서 모임"이라는 카테고리가 가장 쉽게 빠지는 함정 — 어른 흉내 톤 — 에 그대로 빠져 있음.

**추천:**
1. 카피 1차 교체 — "Editorial Brief No. 03", "Now showing", "indie editorial", "— end of brief —", "指讀解" 한자 모두 빼라. "지독해 · 4월" / "이번 달 도착" 같은 한국어 구어로 내려라
2. Instrument Serif italic 사용량 70% 줄여라 — 라벨/eyebrow/숫자 도배는 매거진B 표지 패턴. H1의 `<em>머무는 곳</em>`만 남기고 나머지는 Pretendard 12px Medium uppercase
3. JetBrains Mono를 KPI 숫자에서만, 본문 메타에서 제거 — "오후 7시 · 황리단길 책방"이 Mono로 깔리면 카페 메뉴판 톤
4. 모스 그린 #3D5238을 보조에서 빼거나 더 연하게 — Off-White + Persimmon + Charcoal 3톤만으로 충분. 모스는 status 토큰에서만
5. 배경 도트 그리드 패턴 제거 — 22px 도트는 모눈노트 스킨. 평면 #F7F4EC만
6. 진짜 29CM로 가려면 폰트 위계 뒤집기 — Pretendard 메인, Noto Serif KR은 H1 한 군데 + 인용구만. **매거진B 톤 vs 29CM 톤은 양립 불가능 — 선택해야 함**

### ✍️ 정유진
**핵심 주장:** 매거진 톤은 살아 있지만 영문 라벨 과잉과 인용문 무게감이 어긋나 "독서 모임"이 아니라 "라이프스타일 매거진 광고"처럼 읽힘.

**근거:** 헤드라인 "신청만 하던 곳에서 머무는 곳으로"는 운율 정직해서 좋음. 그러나 한 화면에 `Now showing`, `recent 3`, `join · 24h`, `popular`, `attendance`, `depends · M7`이 동시에 박혀 영문이 디자인 장식으로만 기능 — 회원이 읽을 라벨인지 기획자 메모인지 구분 안 됨. 인용문 "우리는 늘, 자기가 가장 사랑하는 것에 가장 늦게 도착한다"는 너무 잘 다듬어진 아포리즘이라 모임 끝나고 폰으로 칠 문장이 아님 — 작가의 트윗처럼 들림.

**리스크:** "머무는 곳"이라는 약속을 카피가 먼저 깨고 있음 — 영어 라벨 거리감이 한국어 진정성을 밀어냄.

**추천:**
1. **회원용 영문 라벨 제거**: `Now showing` → `이번 주 픽`, `recent 3` → `최근 한 줄`, `join · 24h` → `오늘 +3명`, `popular` → `인기`, `attendance` → `개근`
2. **인용문 교체**: "우리는 늘, 자기가 가장 사랑하는…" → "이 책은 두 번째 읽는데, 첫 번째 읽었을 때 내가 뭘 봤던 건지 모르겠다." 또는 "끝까지 못 읽고 왔는데 다른 분들 얘기 듣고 다시 폈어요." — 첫 줄이 너무 잘 쓰여 있으면 다음 사람이 못 씀
3. **Hero 리드 호흡 정리**: 38자 + 부사절 두 개로 무거움 → "회원은 신청만 하면 떠난다. 운영자는 한 화면에서 모든 걸 한다. Phase 3는 이 둘을 같이 푼다. 회원에겐 머물 이유를, 운영자에겐 숨 쉴 자리를."
4. **버튼 카피 `＋` 제거**: `＋ 나도 한 줄 남기기` → `한 줄 남기기` — `＋`는 어드민 "추가" 기호
5. **섹션 타이틀 톤 통일**: "관리자 데스크톱 — 30초 안에 오늘 뭐 해야 돼?" (구어 의문) → "관리자 데스크톱 — 오늘 할 일을 30초 안에" (다른 섹션의 명사형 매거진톤과 일관)
6. (메모) `Editorial Brief No. 03`, `指讀解` 한자 — 자족적 장식. 회원 노출 화면이라면 빼라

## 라운드 2: 토론

### 상호 반박 & 보강

#### 🎨 김도윤의 반응
- **🧭 박서연 의견에 동의 + 보강:** "병렬 정보가 묻힘"은 정확. 다만 mini Gantt를 추가하기 전에 timeline 자체의 그리드 디시플린이 깨져 있어 도형 추가 시 더 산만해질 위험 — 일단 ms-row의 6-col equal grid를 5-col + 우측 메모 칸으로 재구성하는 게 선결.
- **🎯 이진우 의견에 부분 반박:** 톤이 "매거진B"라는 진단은 정확하지만 "양립 불가능"은 과함. 29CM도 자체 매거진(STORE)에서는 Instrument Serif 인근을 쓴다. 진짜 문제는 톤 선택이 아니라 **선택한 톤을 디시플린 있게 끝까지 끌고 가지 못한 것** — 사이즈/라운드 일관성을 잡으면 같은 톤도 더 동시대적으로 읽힌다.
- **✍️ 정유진 의견에 동의:** 영문 라벨 과잉은 시각적으로도 맞다. 한 화면에 영문 라벨 6종이 떠 있으면 그리드가 잡혀도 톤이 깨진다.

#### 🧭 박서연의 반응
- **🎨 김도윤 의견에 동의:** "거짓 레퍼런스" 리스크는 UX 관점에서도 치명적 — mockup이 토큰 표준이 되어야 하는데 토큰 위반이 30곳이면 후속 구현팀(이 경우 본인)에게 잘못된 신호를 줌.
- **🎯 이진우 의견에 부분 동의 / 부분 보류:** 회원 화면(모바일 mockup) 톤 비판은 100% 동의. 단, 이 문서는 **내부 브리프**이므로 hero/section header 톤은 회원 친화일 필요 없음 — 오히려 의사결정용 톤(밀도 ↑, 숫자 강조)으로 가야 함. **두 톤을 분리해야 한다** — 모바일 mockup 안쪽 = 회원 톤, 그 밖의 문서 chrome = 의사결정 톤.
- **✍️ 정유진 의견에 강하게 동의:** 영문 라벨 제거는 회원 mockup 안에서만 적용. 문서 chrome의 영문은 의사결정 도구로서 OK.

#### 🎯 이진우의 반응
- **🧭 박서연 의견에 핵심 동의:** "두 톤 분리"는 정확한 판단. 회원 mockup 안과 문서 chrome을 같은 톤으로 만든 게 v1.1의 근본 설계 오류. 문서 chrome은 의사결정 도구, 모바일 안쪽은 실제 회원 화면 시뮬레이션 — 둘은 다른 청중.
- **🎨 김도윤 의견에 부분 반박:** "선택한 톤을 끝까지 끌고 가면 된다"는 디자이너 관점이지만, **잘못된 톤을 디시플린 있게 끌고 가면 더 정확히 잘못 도착한다.** 우선 톤 결정부터 — 매거진B로 갈 거면 hero/chrome만, 모바일 안쪽은 무조건 회원 친화 한국어로 분리.
- **✍️ 정유진 의견에 보강:** 인용문 톤 다운은 핵심. 첫 글이 너무 좋으면 진입장벽 = 운영 실패라는 지적은 브랜드 운영 원칙 그 자체.

#### ✍️ 정유진의 반응
- **🧭 박서연 의견에 동의:** "30초 진단이 어렵다"는 카피 관점에서도 맞다 — hero 리드가 38자 + 부사절 두 개로 시간을 잡아먹음. 짧은 두 문장 → 긴 한 문장 리듬이 매거진 표준.
- **🎯 이진우 의견에 동의 + 미세 조정:** 영문 제거 동의. 단 "지독해 · 4월"은 좋은 대안이지만 "이번 달 도착"은 약간 광고 카피 — "4월의 모임"이나 "4월, 이렇게 읽었어요" 같은 더 사적인 톤이 자연스러움.
- **🎨 김도윤 의견에 보완:** 폰트 무게 다운 동의. 한글 명조 700+은 카피에서도 무겁게 읽힘 — 인용문은 무조건 500 이하여야 "진짜 누가 쓴 글" 톤.

### 핵심 논쟁점

   - 박서연·정유진: 톤 선택보다 **두 톤(문서 chrome vs 모바일 안쪽) 분리**가 우선

2. **Phase 3 의사결정 정보의 강조 수준**
   - 박서연: M9·M10 병렬 정보를 hero에 숫자로 끌어올려야
   - 다른 셋: 부분 동의하되, 문서 chrome 톤이 정해진 후 적용 순서

3. **데코 요소(`指讀解`, `Editorial Brief No. 03`, 도트 그리드)의 운명**
   - 이진우·정유진: 자족적 장식, 제거
   - 김도윤: 미학상 제거 OK, 단 grid 디시플린만 잡으면 일부는 살릴 만함
   - 박서연: 의사결정에 도움 안 되므로 제거 동의

### 최종 합의

**합의 (전원 / 강한 동의):**

1. **두 톤을 분리하라** — 문서 chrome(hero, section header, DIFF, timeline)은 **오너 의사결정 도구 톤**(밀도↑, 숫자 강조, 영문 메타 OK), 모바일 mockup 안쪽은 **회원 친화 한국어 톤**(영문 라벨 제거, 인용문 다운, `＋` 제거). 현재 v1.1은 둘을 같은 톤으로 만든 게 근본 오류.

2. **회원 모바일 안의 영문 라벨 6종 즉시 한국어 교체.** `Now showing` → `이번 주 픽`, `recent 3` → `최근 한 줄`, `join · 24h` → `오늘 +3명`, `popular` → `인기`, `attendance` → `개근`, `D-1`은 그대로 OK.
