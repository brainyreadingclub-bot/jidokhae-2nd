## 혼란 유발 지점

### 1. 참가비 단위 불일치 (중)

**현상**: `MeetingDetailInfo`(모임 상세 페이지 정보 카드)에서 참가비가 "10,000"으로만 표시. `MeetingCard`와 `TransferPage`에서는 "10,000원"으로 표시. 동일한 서비스 내에서 금액 단위 표시가 불일치.

**영향 범위**:
- `MeetingDetailInfo.tsx` 95행: `value={formatFee(meeting.fee)}` -- '원' 없음
- `confirm/page.tsx` 120행: `{formatFee(paidAmount ?? typedMeeting.fee)}` -- '원' 없음
- `MeetingActionButton.tsx` 366행 (취소 완료): `{formatFee(cancelResult.refundedAmount)}` -- '원' 없음
- `MeetingActionButton.tsx` 478, 493, 536행 (취소 모달): '원' 없음
- `RegistrationCard.tsx` 66, 74행 (내 신청 카드): '원' 없음

**사용자 영향**: 금액과 관련된 정보에서 화폐 단위가 빠져 있으면 "이게 원인지, 포인트인지" 순간적으로 혼란. 특히 결제/환불 맥락에서 금액 단위는 신뢰감과 직결.

### 2. 404 페이지 미구현 (하)

**현상**: `MeetingDetailContent`에서 `notFound()` 호출 시 Next.js 기본 404 페이지가 표시됨. 커스텀 `not-found.tsx` 파일이 없어서 서비스 브랜딩이 끊김.

**사용자 영향**: 삭제된 모임 링크를 카카오톡에서 클릭했을 때 영문 404 페이지가 나타나면 "사이트가 고장났나?" 느낌. 빈도는 낮지만 발생 시 당황.

### 3. 계좌번호 복사 실패 시 무반응 (중)

**현상**: `BankInfoCard`에서 `navigator.clipboard.writeText()` 실패 시 catch 블록이 비어 있어 사용자에게 아무 피드백 없음. 카카오톡 인앱 브라우저는 Clipboard API를 지원하지 않는 경우가 있음.

**사용자 영향**: 복사 버튼을 눌렀는데 아무 반응이 없으면 "고장인가?" 혼란. 특히 95% 사용자가 모바일 카카오톡 인앱 브라우저라면 높은 확률로 발생 가능.

### 4. 결제 확인 페이지 자동 이동 시간 불일치 (하)

**현상**: `payment-redirect` 에러 시 2초 후 이동, `payment-fail`에서 5초 후 이동. 시간 차이에 대한 명확한 이유 없음. payment-fail에서는 "5초 후 자동으로 이동합니다" 안내가 있지만 payment-redirect에서는 "잠시 후 모임 페이지로 이동합니다..."로 모호.

**사용자 영향**: 크게 불편하지 않으나, 에러 메시지를 읽기도 전에 화면이 전환될 수 있음(2초).

### 5. 결제 완료 확인 페이지에서 금액 혼동 가능성 (하)

**현상**: `confirm/page.tsx`에서 결제 금액 표시가 `paidAmount ?? typedMeeting.fee`로 fallback. `paymentKey`가 없거나 registration 조회 실패 시 meeting fee가 표시되는데, 실제 결제 금액과 다를 수 있음(할인 등 미래 시나리오).

**사용자 영향**: 현재 할인 기능이 없으므로 즉시 문제는 아니지만, 확장 시 주의 필요.

## 특별히 잘된 점

1. **취소/환불 플로우 설계**: 환불 규정 안내 → 금액 확인 → 최종 확인의 3단계 모달은 금전 거래에서 높은 신뢰감을 준다. 환불 0원 시 추가 경고 문구도 적절하다.

2. **대기 신청 UX**: 주황색 버튼으로 일반 신청과 시각적 구분, 하단 설명 텍스트로 동작 원리를 사전 안내. 대기 정보 카드에서 결제 금액과 자동 환불 규칙을 상시 확인 가능.

3. **첫 방문자 온보딩**: 웰컴 스크린 → 프로필 설정의 2단계 게이트가 자연스럽다. "넷플릭스 말고, 할 게 생깁니다" 카피가 서비스 성격을 잘 전달.

4. **빈 상태 처리**: 모임 없음, 신청 없음 등 비어 있는 상태에서 안내 메시지 + 행동 유도 링크("모임 둘러보기")가 일관되게 제공.

5. **비로그인 탐색**: 로그인 페이지에서 "모임 일정 먼저 보기" 링크로 공개 페이지 접근 가능. 결제 없이 모임 정보를 먼저 확인할 수 있어 진입 장벽이 낮다.

6. **법적 요건**: 푸터에 사업자 정보, 이용약관, 개인정보처리방침, 환불정책 링크가 모두 포함. PG 심사 대응으로 auth 레이아웃에도 Footer 포함.

## 종합 판정: 조건부 통과

**판정 사유**: 핵심 사용자 플로우(로그인 ~ 신청 ~ 취소)는 안정적이고 금전 거래 신뢰감도 높으나, 금액 표시에서 '원' 단위 누락이 다수 화면에 걸쳐 있고, 카카오톡 인앱 브라우저에서 계좌번호 복사 실패 시 무반응인 점은 주 사용 환경(모바일 95%, 카카오톡 인앱)을 고려하면 수정이 필요하다.

### 우선 수정 권고

| 우선순위 | 항목 | 설명 |
|---------|------|------|
| P1 | 금액 단위 '원' 통일 | MeetingDetailInfo, confirm 페이지, MeetingActionButton(취소 모달/완료), RegistrationCard에서 `formatFee()` 뒤에 '원' 추가 |
| P1 | 계좌번호 복사 fallback | BankInfoCard에서 Clipboard API 실패 시 계좌번호를 select 가능한 텍스트로 표시하거나, 실패 메시지("복사가 지원되지 않습니다. 직접 입력해주세요") 표시 |
| P2 | 커스텀 404 페이지 | 서비스 브랜딩에 맞는 not-found.tsx 추가 ("모임을 찾을 수 없습니다" + 홈 링크) |
| P2 | 토스트 aria-live | 토스트 컨테이너에 `role="alert"` 또는 `aria-live="polite"` 추가 |
| P3 | label htmlFor 연결 | ProfileSetup 폼 필드에 id/htmlFor 매칭 |
| P3 | 모달 aria-label | ModalOverlay에 `aria-label` prop 추가 |
