Issue #1 — 날짜 필터 해제 발견성
문제 날짜를 선택하면 리스트가 필터링되지만, 사용자가 전체 목록으로 돌아가는 방법을 알 수 없음.
• "오늘" 버튼은 날짜 이동 기능으로 인식됨 (필터 해제 = 숨겨진 기능)
• 같은 날짜를 다시 탭하면 해제되지만, 이것도 발견하기 어려움
• 캘린더를 이리저리 만지다가 "다시 전체 보기"가 불가능해 보임
해결: 필터 활성 칩 (Filter Active Chip) 날짜 선택 시 캘린더와 리스트 사이에 "3월 29일 토요일만 표시 중 ✕" 칩을 표시.
칩을 누르면 즉시 필터 해제. 필터 상태가 항상 눈에 보이고, 해제 방법이 자명함.

UX 검토 종합
잘된 것 • D-day 배지 (오늘/내일/D-N) — 시간 감각을 즉시 제공, 스캔 효율 높음
• 날짜별 그룹핑 — 같은 날짜 카드의 날짜 중복 정보가 DateSectionHeader로 통합
• 캘린더 dot — 모임 있는 날짜/신청한 날짜 시각적 구분 (primary vs accent)
• 헤더 요약 — "이번 달 N회 모임 · 내 신청 N건" 한 줄로 전체 현황 파악
• 카드 좌측 컬러 보더 — 상태(신청완료/모집중/마감)를 스캔 가능한 시각 패턴
• font-display(Noto Serif KR) 적용 — 제목에 서체 감성이 살아남
개선 필요 • 필터 해제 발견성 — 필터 활성 칩 추가 필요 (위 제안대로)
• 카드 내 날짜 정보 중복 — DateSectionHeader가 이미 날짜를 표시하는데, 카드 안에도 날짜 pill이 있음. 같은 날짜 그룹 안에서는 시간만 보여도 충분. (단, 단독 카드일 때는 필요하므로 조건부 숨김 검토)
• 주간 뷰 가로 스크롤 — 5주 분량이 inline-flex로 들어가 있는데, 스크롤 힌트가 없어서 좌우로 넘길 수 있는지 모를 수 있음. 좌우 fade gradient나 스와이프 힌트 고려
추가 제안 (우선순위 낮음) • 캘린더 주간↔월간 전환 시 높이 변화에 부드러운 transition 추가
• 빈 날짜 선택 시 "모임이 없습니다" 대신 필터가 아예 안 걸리게 처리 (현재는 disabled로 막혀 있어 OK)
• 카드의 hover → 모바일에서는 active:scale 축소가 더 효과적 (현재 hover:-translate-y 적용됨)
