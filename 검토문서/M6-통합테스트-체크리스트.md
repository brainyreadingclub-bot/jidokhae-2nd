# M6 통합 테스트 체크리스트

M6 (통합 검증 + 출시 준비) — WP6-1: E2E 검증 + 버그 수정

> **M6 핵심 목표**: 전체 MVP 흐름이 실제로 동작하는지 시나리오 기반으로 검증하고, 발견된 버그를 수정한다.

## STEP 6. 동시 신청 — 정원 초과 (시나리오 6-1-06)

> 정원 1명 모임에 2명이 동시에 결제 → 1명만 성공

### 준비

T4 모임은 정원 **1명**으로 생성되어 있어야 합니다.

### 테스트 방법

1. **브라우저 2개** 준비 (예: Chrome + Edge, 또는 일반 + 시크릿 모드)
2. 브라우저1: member(M1) 로그인 → T4 상세 → "신청하기" 클릭 → 토스 결제창 대기
3. 브라우저2: member(M2) 로그인 → T4 상세 → "신청하기" 클릭 → 토스 결제창 대기
4. **거의 동시에** 양쪽 결제 완료

| # | 확인 항목 | 기대 결과 | 결과 |
|---|----------|----------|------|
| 6-01 | 결제 결과 1 | 1명은 "결제 완료" 확정 | |
| 6-02 | 결제 결과 2 | 1명은 "정원 초과" 안내 + **자동 환불** | |
| 6-03 | T4 모임 상세 | "마감" 뱃지 표시 | |
| 6-04 | Supabase registrations 확인 | confirmed 1건만 존재 | |

> **참고**: 동시 결제가 어렵다면, 한 명이 먼저 결제한 후 다른 브라우저에서 시도하여 "마감" 처리를 확인해도 됩니다.

## 테스트 후 정리

```sql
-- 테스트 모임/신청 정리가 필요한 경우
-- (삭제된 모임은 이미 status='deleted'이므로 목록에 안 보임)

-- 남은 테스트 registrations 확인
SELECT r.id, r.status, r.paid_amount, r.refunded_amount, m.title
FROM registrations r
JOIN meetings m ON r.meeting_id = m.id
WHERE m.title LIKE '%테스트%'
ORDER BY r.created_at DESC;
```
