## STEP 1. Supabase 프로덕션 확인 (시나리오 6-2-01)

현재 사용 중인 Supabase 프로젝트가 프로덕션용인지 확인합니다.

### 1-1. 테이블 확인

Supabase Dashboard → **SQL Editor**에서 실행:

```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
  AND table_name IN ('profiles', 'meetings', 'registrations');
```

**기대 결과:** 3개 테이블 모두 존재

### 1-2. RLS 활성화 확인

Supabase Dashboard → **Table Editor** → 각 테이블 선택 → 우측 상단 "RLS enabled" 뱃지 확인

| 테이블 | RLS 활성화 | 확인 |
|--------|:---------:|------|
| profiles | 필요 | |
| meetings | 필요 | |
| registrations | 필요 | |

### 1-3. DB Functions 확인

SQL Editor에서 실행:

```sql
SELECT routine_name
FROM information_schema.routines
WHERE routine_schema = 'public'
  AND routine_name IN ('confirm_registration', 'get_confirmed_counts', 'is_admin');
```

**기대 결과:** 3개 함수 모두 존재

### 1-4. Trigger 확인

```sql
SELECT trigger_name, event_object_table
FROM information_schema.triggers
WHERE trigger_schema = 'public';
```

**기대 결과:** `on_auth_user_created` (또는 유사 이름) 트리거 존재

### 1-5. 검증 스크립트 실행

로컬에서 실행하여 자동 확인:

```bash
cd jidokhae-web
npm run verify:prod
```

**기대 결과:** 모든 항목 ✅ 통과

| # | 확인 항목 | 결과 |
|---|----------|------|
| 1-1 | 테이블 3개 존재 | |
| 1-2 | RLS 활성화 | |
| 1-3 | DB Functions 3개 존재 | |
| 1-4 | Trigger 존재 | |
| 1-5 | verify:prod 스크립트 통과 | |

## STEP 2. TossPayments 실결제 전환 (시나리오 6-2-02)

테스트 모드에서 라이브 모드로 전환합니다.

### 2-1. 사전 확인

> ⚠️ **사업자등록**이 완료되어야 라이브 모드를 사용할 수 있습니다.
> 사업자등록 전이라면 테스트 모드로 먼저 출시하고, 등록 완료 후 전환할 수 있습니다.

### 2-2. 라이브 키 확인

1. [토스페이먼츠 개발자센터](https://developers.tosspayments.com/) 로그인
2. 좌측 메뉴 → **내 개발 정보** → **API 키**
3. **라이브 키** 탭에서 아래 2개를 복사:
   - **클라이언트 키** (`live_ck_...`)
   - **시크릿 키** (`live_sk_...`)

### 2-3. 웹훅 설정

1. 좌측 메뉴 → **웹훅** → **웹훅 엔드포인트 추가**
2. URL: `https://[프로덕션 도메인]/api/webhooks/tosspayments`
3. 이벤트: `PAYMENT_STATUS_CHANGED` 선택
4. 웹훅 시크릿 키 복사 → STEP 3에서 환경 변수에 설정

> **주의**: 라이브 키로 전환하면 실제 결제가 발생합니다. STEP 6의 실결제 테스트에서 소액(1,000원)으로 확인하세요.

| # | 확인 항목 | 결과 |
|---|----------|------|
| 2-1 | 사업자등록 완료 (또는 테스트 모드로 먼저 출시) | |
| 2-2 | 라이브 클라이언트 키 + 시크릿 키 복사 | |
| 2-3 | 웹훅 엔드포인트 등록 | |

## STEP 3. Vercel 프로덕션 설정 (시나리오 6-2-03)

### 3-1. 도메인 설정

1. [Vercel Dashboard](https://vercel.com) → 프로젝트 선택
2. **Settings** → **Domains**
3. 옵션 A: 기본 `.vercel.app` 도메인 사용 (무료)
4. 옵션 B: 커스텀 도메인 추가 (DNS 설정 필요)

### 3-2. 환경 변수 설정

1. **Settings** → **Environment Variables**
2. **Environment**: `Production` 선택
3. 아래 변수들을 추가/수정:

| 변수명 | 값 출처 | 비고 |
|--------|---------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase Dashboard → Settings → API | |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase Dashboard → Settings → API | |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase Dashboard → Settings → API | ⚠️ 보안 주의 |
| `NEXT_PUBLIC_TOSSPAYMENTS_CLIENT_KEY` | STEP 2에서 복사한 라이브 클라이언트 키 | `live_ck_...` |
| `TOSSPAYMENTS_SECRET_KEY` | STEP 2에서 복사한 라이브 시크릿 키 | `live_sk_...` |
| `TOSSPAYMENTS_WEBHOOK_SECRET` | STEP 2에서 복사한 웹훅 시크릿 | |

### 3-3. 재배포

환경 변수 변경 후 재배포가 필요합니다:

1. **Deployments** 탭 → 최근 배포 → **...** 메뉴 → **Redeploy**
2. 또는 `git push` (코드 변경이 있는 경우)

### 3-4. 접속 확인

프로덕션 URL로 접속하여 페이지가 정상 로드되는지 확인합니다.

| # | 확인 항목 | 결과 |
|---|----------|------|
| 3-1 | 프로덕션 도메인 설정 완료 | |
| 3-2 | 환경 변수 6개 모두 설정 | |
| 3-3 | 재배포 완료 | |
| 3-4 | 프로덕션 URL 접속 → 로그인 페이지 표시 | |

## STEP 4. Kakao OAuth 프로덕션 (시나리오 6-2-04)

### 4-1. 카카오 개발자 콘솔

1. [카카오 개발자 콘솔](https://developers.kakao.com/) 로그인
2. **내 애플리케이션** → 해당 앱 선택

### 4-2. 플랫폼 설정

1. **앱 설정** → **플랫폼**
2. **Web** → **사이트 도메인** 추가:
   - `https://[프로덕션 도메인]`

### 4-3. Redirect URI 추가

1. **제품 설정** → **카카오 로그인**
2. **Redirect URI** 추가:
   - 기존 localhost URI는 유지
   - 프로덕션 URI 추가: (Supabase Auth가 처리하므로 Supabase URL 기반)

### 4-4. Supabase Auth 확인

1. Supabase Dashboard → **Authentication** → **Providers** → **Kakao**
2. **Site URL**이 프로덕션 도메인으로 설정되어 있는지 확인
3. **Redirect URLs**에 프로덕션 도메인이 포함되어 있는지 확인:
   - `https://[프로덕션 도메인]/**`

### 4-5. 로그인 테스트

프로덕션 URL에서 카카오 로그인을 시도합니다.

| # | 확인 항목 | 결과 |
|---|----------|------|
| 4-1 | 카카오 플랫폼에 프로덕션 도메인 추가 | |
| 4-2 | Redirect URI 설정 완료 | |
| 4-3 | Supabase Auth Redirect URL 확인 | |
| 4-4 | 프로덕션에서 카카오 로그인 성공 | |

## STEP 5. Admin 계정 설정 (시나리오 6-2-05)

### 5-1. 운영자(영탁님) 로그인

1. 영탁님에게 프로덕션 URL을 공유합니다
2. 영탁님이 해당 URL에서 카카오 로그인을 완료합니다
3. → profiles 테이블에 자동으로 레코드가 생성됩니다

### 5-2. role 변경

1. Supabase Dashboard → **Table Editor** → **profiles**
2. 영탁님의 행을 찾습니다 (nickname 또는 가장 최근 created_at)
3. `role` 컬럼을 `member` → `admin`으로 변경
4. **Save** 클릭

### 5-3. 확인

영탁님이 페이지를 새로고침하면 하단 탭에 "운영" 탭이 표시됩니다.

| # | 확인 항목 | 결과 |
|---|----------|------|
| 5-1 | 영탁님 카카오 로그인 완료 | |
| 5-2 | Supabase에서 role='admin' 변경 | |
| 5-3 | "운영" 탭 표시 확인 | |
