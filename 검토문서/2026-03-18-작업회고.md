## 🔴 실수: 카카오톡 인앱 브라우저 대응

### 처음에 작성한 코드 (잘못됨)

- 로그인 페이지: 카카오톡 인앱 감지 → "외부 브라우저에서 열기" 배너 추가
- 결제 버튼: 카카오톡 인앱이면 "외부 브라우저에서 결제하기"로 교체

### 왜 잘못인가 — 실제 사용자 흐름을 안 따져봄

1. **카카오톡 인앱 → 카카오 OAuth는 오히려 잘 됨.** 카카오톡 자체 브라우저니까. 로그인 배너는 과잉 대응
2. **"외부 브라우저에서 결제하기" 클릭 → 외부 브라우저에는 로그인 세션이 없음** → 로그인부터 다시 시작. 최악의 UX
3. **이 프로젝트에는 이미 TossPayments Webhook 백업이 구현되어 있음** — 결제 리다이렉트 실패해도 웹훅이 처리
4. React 하이드레이션 불일치 — 서버(navigator 없음=false) vs 클라이언트(카카오톡=true)

### 결과

2건 모두 **롤백**. 로그인/결제 코드는 원본 그대로 복원.
