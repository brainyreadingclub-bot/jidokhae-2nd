---
name: 운영자 워크플로우 우선 파악
description: 기술 설계 전에 운영자의 실제 업무 흐름을 파악해야 — pending_transfer 제거 제안으로 3번 계획 수정
type: feedback
---

운영자의 실제 워크플로우를 먼저 이해한 뒤에 기술 설계를 해야 한다.

**Why:** 계좌이체 흐름에서 "입금 확인 필요 없이 바로 확정"을 제안했으나, 사용자의 의도는 "신청은 바로 되되, 입금 확인은 운영자가 따로 관리"였음. 운영 맥락을 파악하지 않고 기술적 단순화를 성급하게 제안하여 ExitPlanMode가 2번 거부되고 계획을 3번 수정.

**How to apply:** "이 기능을 단순화할까요?"를 제안하기 전에, "실제로 이 기능을 어떻게 사용하시나요?"를 먼저 물어볼 것. 특히 결제/환불처럼 운영자 개입이 있는 흐름에서는 운영 시나리오를 전부 확인한 뒤에 기술 설계.
