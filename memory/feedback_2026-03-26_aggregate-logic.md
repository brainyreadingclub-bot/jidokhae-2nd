# 교훈: 집계 계산에서 "어떤 행이 포함되는가"를 반드시 검증

**날짜:** 2026-03-26
**상황:** AdminMeetingSection의 totalPaid가 confirmed만 합산, 환불은 전체(cancelled)에서 합산 → 순매출 음수
**근본 원인:** 설계 리뷰(검토의견)가 아키텍처 수준이라 계산 로직의 행 범위 불일치까지 잡지 못함

**규칙:**
- 집계 코드(SUM, reduce)를 작성하거나 리뷰할 때: **뺄셈의 좌변(A)과 우변(B)이 같은 행 범위를 대상으로 하는지** 확인
- A - B 패턴에서 A가 subset, B가 superset이면 음수가 나올 수 있음
- 실제 데이터(특히 cancelled가 confirmed보다 많은 모임)로 수치 대입 검증
