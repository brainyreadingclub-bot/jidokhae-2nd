# 교훈: 같은 파일을 수정하는 작업은 편집-커밋을 순차 실행

**날짜:** 2026-03-26
**상황:** 작업 1(0명 표시)과 작업 2(아이콘)가 MeetingDetailInfo.tsx의 인접 라인을 수정. 둘 다 한꺼번에 편집 후 git add로 분리하려 했으나 파일 단위 staging이라 합쳐짐. rebase로 분리 시도 → 다른 커밋(작업 3)이 소실되는 사고 발생.

**규칙:**
- 같은 파일을 수정하는 작업이 여러 개일 때: **작업A 편집 → 즉시 커밋 → 작업B 편집 → 즉시 커밋** 순서를 지킨다
- 전부 편집한 후 사후에 커밋을 분리하려 하지 않는다
- 이미 합쳐진 경우: rebase 대신 커밋 메시지에 두 작업을 모두 명시하고 넘어간다
- `git rebase --onto`는 예상치 못한 커밋 소실 위험이 있으므로 단순 재배치에 사용하지 않는다
