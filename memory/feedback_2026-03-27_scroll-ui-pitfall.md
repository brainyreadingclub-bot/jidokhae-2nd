# 교훈: 가로 스크롤 UI는 스크롤바 숨김 + 상태 동기화를 사전 검증

**날짜:** 2026-03-27
**상황:** CalendarStrip 주간뷰가 5주 가로 스크롤 구조. `scrollbar-hide` CSS가 실제 동작하지 않아 스크롤바 노출 → 사용자가 스크롤바 화살표로 주 이동 → viewDate 미변경 → 월 레이블 불일치. 3번 증분 패치 후 결국 가로 스크롤 제거로 해결.

**규칙:**
- `overflow-x-auto`를 사용하면 반드시 스크롤바가 실제로 숨겨지는지 검증 (Tailwind v4에 `scrollbar-hide` 내장 아님 — 커스텀 CSS 필요)
- 스크롤 위치와 컴포넌트 상태(월 레이블, 선택 상태 등)가 동기화되지 않는 구조는 피한다
- 동기화가 어려우면 스크롤 구조 자체를 단순화 (단일 뷰 + 버튼 네비게이션)
- 증분 패치보다 근본 재설계가 빠를 수 있다 — "왜 스크롤이 필요한가?"부터 질문
